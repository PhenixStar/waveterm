# Releases

Release artifacts are built by CI/CD and published to GitHub Releases. This folder contains release configuration only.

Artifacts (installers, binaries, ZIPs) are never committed here — they are generated by the build pipeline and uploaded directly to GitHub Releases.

## Release Process

See [RELEASES.md](../RELEASES.md) for the full step-by-step release workflow.

## Output Artifacts (Windows)

Built by `task package` → placed in `make/`:

| File | Type |
|------|------|
| `Terminolgy-win32-x64-{version}.exe` | NSIS installer |
| `Terminolgy-win32-x64-{version}.msi` | MSI installer |
| `Terminolgy-win32-x64-{version}.zip` | Portable ZIP |
